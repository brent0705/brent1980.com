[ career.log ]
// early phase
系統開發與維運基礎

從 ERP 系統維護入門。接著是 MES、生產線整合、Oracle 資料庫、SQL Server。 那段時間沒有什麼大專案，就是修、維護、讓系統不要壞掉。 但也因為這樣，我知道了一件事：穩定比功能重要。 一個沒人注意到的系統，往往才是最成功的系統。

// growth phase
企業系統整合

開始接觸跨系統的工作。POS、Flow、ERP——每套系統都有自己的邏輯、自己的格式、自己的歷史包袱。 我的工作是把它們串起來，讓資料能流動，讓一邊的動作能讓另一邊知道。 這段時間讓我理解：系統整合不是技術問題，是流程問題。 你必須先搞懂業務在怎麼運作，才知道資料應該怎麼走。

// current
系統整合與維運

現在做的事情範圍很廣。ERP 維運、地磅硬體串接、跨系統資料同步、API 架構設計、 自動化流程建置、報表分析系統、智慧倉儲整合。 這逾 20 年的積累，讓我不只是在寫程式——而是在讓一整個系統環境持續運作。 不是等問題發生才處理，而是在問題發生前把它排除。

[ work.style ]
[ approach ]
  • 先理解，再動手 — 接到一個需求，我不會馬上開始寫。我會先釐清問題的本質，弄清楚現有流程的樣子，然後才決定怎麼做。這讓我少走了很多冤枉路。
  • 穩定優先 — 我不追求最新的技術框架。我追求的是：這套系統三年後還能正常跑，換了人維護也不會出問題。技術選型的標準是可維護性，不是潮流。
  • 從整體看問題 — 單一功能的問題，往往根源在流程設計。我習慣拉高視角，從整體系統架構去理解一個局部問題，找到真正的根因。
  • 能溝通的工程師 — 資訊部門的工作，有一半是跟人的工作。我能用非技術語言跟業務、生產、管理層溝通，把他們的需求轉化成系統方案，也能把技術限制說清楚讓他們理解。
