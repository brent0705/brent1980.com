關於我

從開發工程師到資訊主管，16 年專注企業系統的實務歷練

Brent｜資訊部副理

擁有 16 年以上企業資訊系統開發與管理經驗。從基層系統開發工程師做起，歷經 ERP 維護、MES 開發、POS 系統整合、內部平台建置，一路成長為帶領資訊部門的管理角色。

我擅長的不只是寫程式，而是站在系統架構的高度，理解企業營運流程，將業務需求轉化為穩定、可維護的系統方案。多年來的實務經驗讓我深知：好的系統不是功能堆疊，而是對問題本質的深刻理解。

在技術面，我以 C#/.NET 與 MSSQL 為核心，涵蓋桌面應用、Web 開發、API 設計、報表自動化與系統整合。在管理面，具備部門策略規劃、預算編列、跨部門溝通協調的完整歷練。

目前職務 — 資訊部副理

身為資訊部門副理，我負責部門的整體規劃與執行，主要工作涵蓋：

  • ERP 系統維運與持續優化，確保核心營運系統穩定運行
  • 磅單系統開發與地磅資料串接，實現過磅作業數位化
  • 資料同步機制設計，確保多套系統間資料一致性
  • API 同步架構設計與建置，串接內部與外部系統
  • 內部自動化流程建置，包含排程報表、異常通知、資料匯整
  • 資料報表與營運分析系統開發，支援管理層決策
  • 智慧倉儲系統整合，包含條碼管理與設備串接
  • 企業內部工具開發，針對各部門痛點提供對應方案

專業歷程

早期 — 系統開發與維運基礎

從 ERP 系統維護入門，累積 MES 系統開發、生產系統整合的實務經驗。在 Oracle 與 SQL Server 環境下，扎實打下資料庫管理與後端開發的技術基礎。

成長期 — 企業系統整合

接觸 POS、Flow 等多套企業系統的維運與開發工作。開始獨立負責 API 串接、內部平台建置與自動化報表系統，逐步建立跨系統整合的完整能力。

管理期 — 資訊主管角色

擔任資訊部門主管，負責部門策略規劃、年度預算編列、系統整合架構設計。除了技術面，更需要與業務、生產、管理層密切溝通，確保資訊投資對準企業營運目標。
